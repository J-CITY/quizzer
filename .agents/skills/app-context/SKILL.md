# Контекст проекта Quizzer (Agent Context)

## Описание
Quizzer — это мобильное приложение на Flutter для изучения японских слов (и других языков) при помощи системы карточек с 4 вариантами ответов. 
Слова можно загружать напрямую из публичных Google Таблиц, тренировать их с помощью умной системы прогрессии, а также отслеживать свою активность и стрик.

## Стек технологий
- **Фреймворк**: Flutter (со включенным Material 3)
- **Стейт-менеджмент / DI**: `flutter_riverpod`
- **Локальная база данных**: `isar`
- **Фоновые задачи (уведомления)**: `workmanager`, `flutter_local_notifications`
- **Аудио и TTS**: `flutter_tts` (озвучка слов), `audioplayers` (звуковые эффекты правильного/неправильного ответа)
- **Локализация**: Стандартный `flutter_localizations` (через `.arb` файлы: `app_ru.arb`, `app_en.arb`)
- **Графика и графики**: `fl_chart` (статистика по дням и прогрессу слов), `lottie` (анимации стриков, динамическая перекраска)

## Структура проекта
- `lib/data/models/`: Модели Isar:
  - `Word` (слово: id, японский, чтение, перевод, прогресс от 0 до 5, дата последней тренировки).
  - `CustomList` (пользовательские списки слов: название, опциональный `googleSheetId`, список слов `IsarLinks<Word>`, флаг синхронизации при запуске).
  - `Settings` (настройки приложения: звуки, TTS, время уведомлений, типы вопросов).
  - `TrainingSession` (история тренировок для статистики и подсчета дней подряд).
- `lib/data/services/`: 
  - `DatabaseService` (управление БД Isar).
  - `GoogleSheetsService` (парсинг публичных Google Таблиц в формате CSV).
  - `NotificationService` (управление фоновыми напоминаниями потренироваться с учетом разрешенного интервала "С: До:").
- `lib/domain/`:
  - `TrainingEngine` (бизнес-логика формирования сессии вопросов: выборка слов (режимы "Изучение" и "Повторение"), генерация вариантов ответов, перемешивание).
- `lib/ui/`: Пользовательский интерфейс (экраны: списки, тренировка, результаты, статистика, настройки).
- `lib/utils/`: Вспомогательные утилиты, включая `LottieColorShift` для парсинга и изменения цветов анимаций Lottie "на лету" (замена HSL/цветов).

## Основные фичи и логика
1. **Интеграция с Google Sheets**: 
   Пользователь вставляет ID таблицы. Приложение вытаскивает название (с умным парсингом окончания " - Google") и скачивает слова из первых трех-четырех колонок: Японское слово, [Чтение (хирагана/катакана)], Перевод.
2. **Логика тренировки**: 
   - Два режима: "Учить" (берет слова, которые еще не выучены) и "Повторить" (только те, что уже затронуты).
   - Во время тренировки слово озвучивается TTS *после* того как дан ответ (если включена авто-озвучка), либо по нажатию на кнопку динамика.
   - Проигрываются звуки `true.mp3` и `false.mp3` (из `assets/sounds/`).
   - Ошибочные ответы скидывают прогресс слова в 0 и вопрос переносится в конец списка текущей сессии для повторного закрепления.
   - Правильные ответы повышают прогресс до максимума (5).
   - При попытке преждевременно выйти с помощью кнопки Назад или свайпа, показывается подтверждение (`PopScope`), предотвращающее потерю прогресса сессии.
3. **Статистика и Стрики (Окно Статистики)**:
   - Подсчитывается непрерывный "стрик" (дни подряд). На главном экране выводится анимированный огонек Lottie. Цвет огонька динамически перекрашивается в зависимости от величины стрика с помощью самописного парсера JSON (`LottieColorShift`).
   - Отображаются графики тренировок по дням в выбранном месяце (`fl_chart` BarChart).
   - Отображаются круговые диаграммы прогресса по выученным словам — как общие, так и для каждого списка в отдельности (`fl_chart` PieChart).
4. **Уведомления**:
   Используется `workmanager`, который периодически просыпается в фоне, проверяет, попадает ли текущее время в разрешенный интервал (например, с 10:00 до 22:00), и если да — пушит локальное уведомление.
5. **Списки**:
   В приложении можно создавать свои локальные списки, перекидывать туда слова из других списков, сбрасывать прогресс всего списка, либо привязывать список напрямую к Google Таблице (с возможностью синхронизации данных). По нажатию на слово в списке выезжает BottomSheet с информацией.

## Важные команды для работы
- Перегенерация Isar моделей: `dart run build_runner build -d`
- Перегенерация локализации: `flutter gen-l10n`
